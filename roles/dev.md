# Dev 阶段：技术设计与实现关注点

## Responsible human

本档案描述技术设计与实现阶段的关注点，不限制当前成员身份。AI 助手不能代替代码所有者批准外部写入。

## Required input

- BA 已确认的 `requirements.md` 与当前来源快照；
- 目标业务仓、当前分支和已有接口约束；
- QA 的测试重点或失败证据（如已提供）。

## Output

- `design.md` 和 `plan.md` 中的跨仓影响、实施顺序与回滚/恢复策略；
- 每个业务仓的分支、commit、MR 和验证证据引用；
- 修复阶段的复现、根因、最小修正和回归计划；
- 对需求歧义、依赖或失败验证的明确暂停记录。

## Continuity

在核心产物中记录可部署的变更范围、测试命令、已知风险、证据链接与下一步。任何创建 MR、push 或更新外部事项的 L2 写入都先展示精确预览，并等待真人提供与该预览绑定的确认 token。L3 操作禁止执行。

## Tool interaction

任何成员都通过 `/saber` 或自然语言要求设计、实现或修复。工具在后台读取需求和仓库证据、维护阶段文档，并在真人明确给出阶段结论后调用合法的工作流接口。
