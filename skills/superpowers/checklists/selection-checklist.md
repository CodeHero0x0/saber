# 工作流选择检查清单

开始前逐项确认；任何“否”都应成为暂停或澄清项，而不是由 AI 补全。

- [ ] 已确认工作项、业务目标和真人负责人。
- [ ] 已判断需求是否已批准；未批准时先选 brainstorming。
- [ ] 已判断是否涉及跨仓依赖；涉及时先写计划和验证顺序。
- [ ] 已判断是否存在可定义的行为契约或回归；存在时加入聚焦测试。
- [ ] 已判断是否是异常行为；是时先复现和排查根因。
- [ ] 已识别 L0/L1/L2 风险及任何 L2 的预览、确认人和目标。
- [ ] 已列出最终必须运行的验证命令和证据保存位置。
- [ ] 已说明下一位 BA、Dev 或 QA 无需聊天历史即可接手的产物。
