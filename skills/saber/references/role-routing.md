# Saber 角色路由参考

本表只帮助选择工作上下文，不授予身份、审批权或外部写权限。用户显式指定角色或动作时优先采用；若动作会跨过当前工作项门禁，应说明冲突并等待责任人处理。

| 工作信号 | 路由上下文 | 需要的证据 |
| --- | --- | --- |
| 新需求、范围、验收标准、BA 澄清或最终验收 | BA | 已确认来源草稿、需求和真人验收决定 |
| 设计、计划、实现、评审修复或 QA 退回 | Dev | 设计/计划、仓库引用、提交或 CI 证据 |
| 测试、回归、质量结论 | QA | 可复现步骤、测试输出、缺陷与回归证据 |
| 工作项处于暂停或来源漂移 | 当前责任角色 | 恢复原因、新指纹和真人决定 |

无法读取工作项、状态无责任角色或用户意图有两个不同结果时，返回已知事实并只问一个会消除歧义的问题。不要用当前物化默认角色覆盖已有工作项的责任角色，也不要用角色名称推导授权。
