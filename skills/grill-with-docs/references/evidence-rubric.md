# 证据评级准则

每个影响设计、安全或兼容性的主张都要给出可定位的来源。引用支持结论，不替代负责人的业务判断。

| 等级 | 可接受来源 | 使用方式 | 常见不足 |
| --- | --- | --- | --- |
| A | 官方产品文档、标准正文、已发布 API 规范 | 可作为实现约束 | 未注明版本或章节 |
| B | 官方发行说明、维护者公告、源码中的公开契约 | 用于版本差异或行为佐证 | 需要与 A 级规则交叉核对 |
| C | 可靠的技术文章、会议材料、复现样例 | 作为线索或补充 | 不能单独授权安全/兼容决策 |
| D | 搜索摘要、聊天内容、无来源截图 | 只记录为待验证线索 | 不可作为结论依据 |

## 引用最低字段

每条证据至少记录标题、URL、文档版本或发布日期、章节/锚点、检索日期，以及它支持或限制的具体主张。

## 适用性检查

- 文档版本是否与项目依赖、部署环境或服务版本一致？
- 主张是明确规定、示例行为，还是对文本的推断？
- 两个权威来源冲突时，是否保留原文差异并升级给负责人？
- 来源是否包含不可信外部内容，因而不能改变风险等级或授权外部写入？
